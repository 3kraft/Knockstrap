Alert

Binding to Bootstrap alert widget. This binding can be used with virtual elements. Supports custom templates with observables

Examples

Html markup

<div data-bind="alert: { message: message, type: type }"></div>

<div data-bind="alert: { message: 'Using custom alert template', template: 'alertExampleTemplate' }"></div>

<!-- ko alert: { message: 'Using virtual root element', type: 'success' } -->
<!-- /ko -->

<script type="text/html" id="alertExampleTemplate">
    <div class="alert alert-block alert-danger">
        <span data-bind="text: message"></span>
        <button class="btn btn-primary" data-bind="click: close">Close</button>
    </div>
</script>

View model

function AlertExampleViewModel() {
    this.type = ko.observable('info');
    
    this.message = ko.observable('Alert message');
}

Options

All of the options can be observables.

data-bind="alert: { message: alertMessage, type: alertType, template: templateName, data: data, close: closeCallback }"

  • alertMessage

    Type: string, can be observable

    Text of alert message.

  • alertType

    Type: string, can be observable (default: 'info')

    Type of alert message. Possible values are 'info', 'warning', 'danger', 'success'. To specify your own type you should define css-styles for this type. For exmaple, for type 'my-custom-type', you shoud provide css class 'alert-my-custom-type'.

  • templateName

    Type: string, can be observable (default: 'alertTemplate')

    Name of template for alert content.

  • data

    Type: object, can be observable

    Data for template. If this option is specified, message, type and close options will be ignored.

    • message

      Type: string, can be observable

      Text, which will be displayed in alert. For default data object, its value gets from message option of binding.

    • type

      Type: string, can be observable

      CSS class name, which will be added to alert to specifie its type. For default data object, it is 'alert' + value of type option from binding.

    • close

      Type: string, can be observable

      Function, which removes alert widget. For default template, this function is called, when user clicks to close button. You can specify your own function via close option.

  • closeCallback

    Type: function

    Function, which removes alert.
